    原则Principles

    《原则Principles》清单

    • 塑造良好的公司文化
    • 相信真相，真相没什么可怕的。
    • 创造这样一种氛围：只要是合理的事情，人人都能相互理解，没有人有权妄加批评，除非他能开诚布公地讲他的理由。
    • 要极为开放，极为透明。
    • 保持正直，要求别人也必须保持正直。
      • 若当面不对别人进行评论，背地里也不要说。若未曾当面控诉过别人，也不要背地里试探。
      • 所谓的“忠诚”不能成为真相与开放的拦路虎。
    • 记录每一次会议，并分享给与之有关系的人。
    • 绝不容忍失信。不要相信失信之人说他已痛改前非绝不再犯。
    • 创造这样一种文化：容许犯错，并从错误中进行识别、分析、吸取教训。
    • 要意识到处事高效创新的思考者都会犯错。
    • 不要为自己或别人犯的错而郁郁寡欢，要热爱这些错！
    • 仔细观察所犯错误的模式，看看它们是不是自身缺点所导致。
    • 不要因为自己或别人的缺点而感到糟糕。
    • 别老担心面子上过不过得去，而要担心是否会影响目的的实现。
    • 别去管“责备”或“赞扬” ，要习惯关注错误的归因是“精准”还是“不精准”。
    • 错误归因要具体到个人。
    • 写下你和别人的缺点，帮助自己牢记并承认这些缺点。
    • 若因犯了错而感到痛苦，记得要反省。
    • 要自我反思，也确保你身边的朋友们也都是懂得自我反思的。
    • 教导和强化基于错误学习的优点。
      • 最有效的工具是建立“问题日志”，旨在鉴别问题，并从中吸取教训。
    • 不断争取意见统一。
    • 要去伪存真，争取在解决方案上的意见统一。
    • 讨论“这是对的么？”和“这事有意义么？”
    • 认为对的事情，要据理力争。
    • 既要立场坚定，也要保持兼容并蓄。
      • 问问自己，是否拥有发表观点的权利。
      • 要知道自己始终拥有提问的权利。
      • 区分思维开放的人和思想保守的人。
      • 思想保守、缺乏经验之人，还是敬而远之吧。
      • 谨防纸上谈兵、夸夸其谈的人。
      • 谨防不知为耻的人。
    • 确保主要责任方对于他人提出的问题与评论都是持开放态度的。
    • 冲突对于建立重要关系是大有裨益的，因为通过冲突人们才能确定对方的原则是否与自己的一致，便于化解分歧。
      • 分歧中蕴含巨大潜力，尤其对于两个有想法的人之间的分歧而言。
    • 知道什么时候终止辩论，进而讨论一致的解决方案。
      • 当有人质疑就某事展开辩论的重要性时，辩论是有必要的。
      • 办法总是问题多。
      • 要想通过分歧获得积极的结果，评估个人抉择或决策者的观点时需要树立大局观。
      • 要区分两个概念：1）无用的抱怨 2）旨在实现改善的合理诉求。
    • 赞赏开放式辩论的讨论方式并不意味着要通过全体投票来制定规则。
    • 要评估事项是否需要辩论、讨论或传授。
      • 为了避免产生误解，需要确定使用何种沟通方式 （辩论、讨论或传授）。
      • 旨在获得最佳方案的沟通，应该邀请最相关的人参与其中。
      • 旨在教育，增强凝聚力的沟通，如果目标是获取最优解决方案，那么就应该听取更多人的意见。
      • 充分利用各种沟通手段。
    • 不要对所有的观点?视同仁。
    • 思考自己和别人的“可信度”。
      • 问问自己，是否拥有发表观点的权利。
      • 若有人多次成功解决悬而未决的问题，面对质疑也能讲得头头是道，这种人的观点最可信。
      • 如果有人问你一个问题，首先要考虑自己是不是能够解答这个问题的人。
    • 为了“意见统一” ，花再多的时间与精力都不为过，因为这是最有价值的投资。
    • 如果是你主持会议，请协调好会议中各方的讨论。
      • 弄清会议的服务方和主持方。
      • 根据会议目标与重点议题，确定会议的交流方式。
      • 主持讨论要坚定自信，开诚布公。
      • 组织三至五人的小组讨论，邀请思维灵活、概念清晰的成员开放地寻求最佳方案，这种情况一般能取得最好的效果。
      • 要明确讨论的层次的方向。
      • 注意不要让讨论偏题。
      • 增强沟通的逻辑性。
      • 实质内容比形式更重要。
      • 在讨论中要得出一定结论。
      • 安排人做会议纪要，保证会议讨论的事项后续落实。
      • 需要注意的是，集体决策时不要忘记了个人的职责。
    • 不要将控诉、献言献策、辩论的权利同抉择权混为一谈。
    • 要认识到，达成意见统一是双向责任。

    • 如果意见无法统一，提交上级进行讨论决定。

    • 选对人，用对人

    • 要知道，最重要的选择是选谁做负责人。
    • 几乎所有的成功都是来自优秀的文化以及在其中工作的优秀的人才。
    • 首先，要选择合适的人参与规划。
      • 最为重要的是要找同你价值观一致的人。
      • 要找愿意客观评价自己且有自身性格特点的人。
      • 根据需要完成任务和实现目标，选择具备相关概念性思维和常识的人履职尽责。
    • 责任人是需要承担一切后果的。
    • 总的来说，日积月累，你会得到你应得的。
    • 最重要的责任人要为目标、结果和组织机构负主要责任 （即位于组织里的上层的人）。
    • 选择那些明白“ 目标”与“任务”之间差异的人来做事。
    • 要知道每个人生来和后天塑造都是不同的。
    • 考虑他们在价值观、能力和技能上的差异。
    • 要了解你每个员工的情况，才能知道你能在他们身上有何种期待。
    • 岗位用人要与职位要求相匹配。
    • 看待事物与思维方式不同的人，在交流和相处上是存在困难的。
    • 雇佣对的人，用错了人，代价会极为惨重。
    • 仔细审度自己想要的员工应具备什么样的价值观、能力和技能。
    • 招聘员工时，要多考虑员工的价值观与能力，这比技能更重要。
    • 寻找可提出好问题的员工。
    • 要选熠熠生辉之人，而不是又一个平庸之辈。
    • 以人论酬，而不是以岗论酬。
    • 要知道无论你在招聘人才方面有多在行，你所聘用的人选都很有可能不是岗位的最佳人选。
    • 像设计和运行一台机器一样做好管理式作，才能实现预期目标。
    • 明白管理、微观管理和不管理之间的区别。
    • 不断比较完成情况和目标之间的差距。
    • 在所管理的机制内，从更高层次审视自己和机制。
    • 解决问题要参照同类别问题解决时所遵循的原则。
    • 问题出现时，要展开两个层面的讨论：
      • 从机制层面来讨论，为什么会出现这个问题。
      • 从问题本身层面来讨论，当下应如何解决。
    • 和员工的关系不是服从与被服从，而应是相互理解。
      • 不要试图给员工下命令控制他们。
      • 沟通要有逻辑，多听取反馈意见。
    • 明确责任分工。
    • 对员工进行工作问责制度，若他们问责你，要感谢他们。
      • 要分清楚，有些工作上的失败是因为员工没履行一开始的“约定” ，而有些则是因为一开始就没有指定“约定”。
    • 避免“上级卷入下级工作职责”现象
      • 提防那些混淆目标与任务的员工，不能理解工作目标的员工是不值得信任的。
    • 要有主人翁思维，并希望员工们也能具备这种思维方式。
    • 必要时“强迫”自己和员工迎难而上。
      • 对自己和他人要采取问责制度。
    • 别担忧员工喜不喜欢你，还是多想想自己所做的事情能不能帮员工和团队获得成功吧。
    • 知道自己想要什么，坚信自己认为的是正确的事情，不要轻易被人牵着鼻子走。
    • 计划沟通要清晰明了。
      • 已达成共识的目标任务要让所有相关人员都知道。（包括有关部门的员工及监管层的领导）
      • 要警惕交流中出现没有重点，低效无用的句式：我们应该如何。
    • 保持与员工意见统一。
    • 避免和员工产生距离感。
      • 工具：使用每日进度更新了解员工工作与思考的情况。
    • 对员工的信任度不能先入为主，要逐步去了解。
    • 处理问题时，根据自己感觉有把握的情况来调整参与度。
    • 避免说“理论上应该”。
    • 决策中最重要的是讲究逻辑、给明理由且符合常识。
    • 尽管做决策时主要依靠逻辑，但情感感觉也是很重要的。
    • 如果发现自己无法有效解决问题时，应将问题提交给上级，确保为你工作的员工们也是这样操作的。
      • 确保员工主动积极。
      • 工具：升级按钮。
    • 跨部门间或跨子部门间出现问题时，需要上级部门负责人参与定夺。
    • 不要让人逃避责任。
    • 不要想当然地认为人们的答案都是正确的。
    • 将调查透明化，不要私下进行。且是准确地，而不是善意地评估员工。
    • 以同等的严厉程度来评价员工、雇佣新人。
    • 充分了解员工的特质，因为人才是最重要的资源。
    • 要认识到，尽管大多数人都喜欢被表扬，不喜欢被批评，但准确的批评却是最宝贵的。
    • 提供频繁的、清楚的、诚实的反馈，并鼓励就这些反馈进行讨论。
    • 你以及你管理的人都会经历个人成长的过程。
    • 关注个人行为模式，但不要过分解读任何单一事件。
    • 不要认为擅长（或不擅长）某件事的人就一定对所有的事都擅长（或不擅长）。
    • 如果某人工作做得很差劲，要去想这是因为缺少学习 （培训或相关经验）还是因为缺少能力。
    • 在评估员工时最常犯的两个错误，不要犯这些错误。
      • 对于评估结果过于自信；
      • 对于评估结果意见无法达成一致。
    • 在评价员工时，并不需要达到“不含一丝质疑”的程度。
    • 通过实战经验来培训、测试员工。
    • 培训对个人的成长过程起着指引作用。
    • 每件事都是一个案例
    • 授人以鱼不如授人以渔。
    • 有时让人犯错并从中吸取教训要比直接告诉他们一个更好决定更明智。
      • 在批评时，要提一些有建设性的意见。
      • 从成功中学习，也要从失败中学习。
    • 分清哪些错误是可以接受的，哪些是不能接受的。不要让员工犯不可接受的错误。
    • 一般需要18个月持续不断的巩固才能实现行为矫正。
    • 培训员工，而不是改造员工。
      • 常见错误：在没有对一个工作表现差的员工进行能力评估之前，培训、测试该员工，试图让他获得所需技能。
    • 在找到真相之后（即了解到员工的真实情况），谨慎思考下一
    • 一旦发现某人不能胜任工作，尽快将其调离该职位。
    • 勉强将某人留在不合适的职位上要比将其开除更糟糕。

    • 不要降低标准。

    • 发现、诊断、解决问题

    • 懂得如何有效地发现问题，发现问题是优秀管理的第一步。
    • 问题是提升的动力。
    • 你需要观察事情是在预期之上还是在预期之下，确保你的员工也有此观察力。
    • 不要容忍问题。
    • 找尽可能多的人手来一起发现问题。
    • 不要使用模糊的人称，如“我们”或“他们” ，这样做会掩盖个?责任。请使用具体人名。
    • 具体问题要具体对待，不要一开始就过于宽泛。
    • 使用以下工具来捕捉问题：问题日志、计量图表、调查问卷、清单、外部咨询以及内部审计。
    • 最常见的无法观察到原因的问题是被称为“温水煮青蛙”的问题。
    • 在某些情况下，因为某些问题实在难以解决，人们不得不接受那些不可接受的问题。但是，解决那些不可接受的问题其实要比不解决它们更容易，因为不解决它们，将后患无穷。
    • 已经拥有有效周密的解决方案的问题与那些没有解决方案的问题天差地别。
    • 要认识到所有问题只是其根本原因的表征，所以要通过诊断分析来理解问题症结所在。
    • 不要试图从一个“点”中获取大量信息，而应该快速压榨大量的“点” ，并将它们相互联结，从而形成更丰富的图像。
    • 要区别才能问题和能力问题，就去想象如果一个人有足够的才能，那他在这个特定职能上的表现会是如何。
    • 不要做事后诸葛。
    • 找出违背了哪些原则。
    • 前进之前请先回顾。
    • 三思而后行。花点时间做计划。
    • 你拟出的组织设计应该能最小化问题，最大化机会。
    • 使自己体会一段时间的“痛点” ，你就会更理解自己的设定针对的是什么样的对象。
    • 以目标为中心建立机构，而不是以任务为中心，这是重中之重。
      • 首先，设计最佳工作流，在一个组织图中画出草图，将各部分互动情况形象化，标出每个职位所需的特质，最后，选择合适的员工来填充岗位 (根据他们的能力和意愿来进行需求匹配) 。
      • 按照最富逻辑的组团方式来组建部门和子部门。
      • 让每个部门尽可能的自给自足，以此确保他们能够自主控制达成目标所需的资源。
      • 公司效率的下降与官僚作风的扩张程度与公司人数增长和复杂性提升直接相关。
    • 经常思考该如何让事情发挥最大的效果。
      • 将细节工作委派给他人。
      • 与其让一众能力平庸之人获得不那么精良的装备，不如只给一小部分聪明人配备最好的技术。
      • 使用执行力强的人。
    • 谨防“部门职能错位”和“职责错位”。
    • 在分配责任时，注意考虑工作流的设置和员工的能力，而不是岗位头衔。
    • 谨防过分依赖外部咨询。
    • 工具：使用流程手册。
    • 使用任务清单。
      • 不要将任务清单与个人责任混为一谈。
      • 系统性并不意味着必须全部由电脑来控制。
      • 要“重复工作” ，不要“重复检查” ，保证重要任务完成无误。
    • 记住，如果没有警察，法律则形同虚设。
    • 按计划进行。

    • 坚持到底！

    • 有效决策

    • 你的目标是找到最佳答案，而找到最佳答案的可能性是很小的，就算你真的找到了，你也无法确信自己成功了，你必须让其他的靠谱的人来对你进行测试。
    • 处理未知的能力要比知道某事的能力更强大。
      • 鼓励提问：“有什么是我不知道的呢？那我该怎么办呢？”
      • 提出正确的问题与获得正确答案同等重要。
    • 你的目标是寻找最佳答案，而不是在已有的答案中挑一个最好的。
    • 每个人都有权拥有自己的问题和理论，但是只有靠谱的人才有权提出观点。
    • 时刻警惕考虑不周的情况。
      • 征求别人的批评意见，并看到批评的价值。
      • 吸收众人观点。
    • 做决定要讲逻辑，基于期望值测算。
    • 考虑结果的可能性与收益，确保不可接受结果的可能性为零。
      • 不良决策的代价等同于甚至严重于正确决策带来的回报，因此，知道自己没掌握什么，至少和知道自己掌握什么一样有价值。
      • 鉴别出有亏少利多的时机，就算获利可能性低也要试试。
      • 精准评估，提高决策的准确性十分有价值。
      • 任何事情都不能押过多赌注，要留15%或更多的余地给无关联的赌注。
    • 牢记80/20法则，并知道那关键的20%是什么。
    • 区分重要事项和不重要事项，先处理重要事项。
      • 不要做完美主义者。
      • 所以做决定时，最重要的事项是比较少的（一般少于五件）。
      • 警惕“细节焦虑”。
      • 不要混淆小事情和不重要事项，因为小事情也可能很重要。
    • 确保所有“必须完成的任务”在完成时优先于其他任何事情。
    • 最佳选择是基于赞同意见多于反对意见的，当然不是说不允许任何反对意见。要警惕有人在不恰当权衡正 反意见的基础上挑错，进而反对。
    • 要警惕在没有考虑所有可能性的情况下，就低效地确定其可能性，这样会打乱优先次序。
    • 理解并串联关键点
    • 知道可接受的改善速度是多少，因为改变的程度比改变的速度更关键。
    • 如果最佳方案不够令人满意，就得尽力想出更好的办法，实在想不到就提交给上级。
    • 经受住诱惑，避免让不可妥协的事情得到妥协。
    • 不要试图让所有人都满意。
